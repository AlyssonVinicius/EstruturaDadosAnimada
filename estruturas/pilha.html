Uma pilha é uma estrutura de dados linear que segue o princípio LIFO (Last In, First Out), o que significa que o último elemento adicionado é o primeiro a ser removido. Uma analogia comum é pensar em uma pilha de pratos, onde você adiciona um prato no topo e retira do topo também.


 Características Principais da Pilha:

 - Empilhar (Push): Adicionar um elemento ao topo da pilha.

 - Desempilhar (Pop): Remover o elemento do topo da pilha.

 - Topo (Top): O elemento no topo da pilha.

 - Tamanho: O número de elementos na pilha.

 - Vazia (isEmpty): Verificar se a pilha está vazia.

 - Cheia (isFull): Em algumas implementações, verifica se a pilha está cheia (pode ser relevante em pilhas com tamanho fixo).


 Operações Básicas:

 - Empilhar (Push): Adiciona um elemento ao topo da pilha.

 - Desempilhar (Pop): Remove o elemento do topo da pilha.

 - Topo (Top): Obtém o elemento no topo da pilha sem removê-lo.

 - Vazia (isEmpty): Verifica se a pilha está vazia.

 - Cheia (isFull): Verifica se a pilha está cheia (em algumas implementações).


 Implementação:

 Assim como uma fila, uma pilha pode ser implementada usando uma estrutura de dados como um array ou uma lista encadeada. Em linguagens de programação orientadas a objetos, é comum criar uma classe para representar uma pilha, e as operações básicas podem ser métodos dessa classe.


 Aplicações de Pilhas:

 - Chamada de Funções: A pilha é usada para gerenciar a execução de funções em muitas linguagens de programação. Cada chamada de função é empilhada, e quando a função termina, ela é desempilhada.

 - Expressões Matemáticas: Pilhas podem ser usadas para avaliar expressões matemáticas, convertendo notações infixas para postfixas e, em seguida, avaliando as expressões postfixas.

 - Desfazer (Undo) em Aplicações: Pilhas são frequentemente usadas para rastrear operações que podem ser desfeitas em softwares.


 As pilhas são uma estrutura de dados fundamental em ciência da computação e são amplamente usadas em diversos contextos. Elas fornecem uma maneira eficiente de gerenciar dados onde o último elemento adicionado é o primeiro a ser removido.
