O Merge Sort (ou ordenação por junção) é um algoritmo de ordenação eficiente e estável que utiliza a abordagem de "dividir para conquistar" para ordenar elementos em uma lista. Ele divide repetidamente a lista em sub-listas menores, ordena essas sub-listas e, em seguida, combina-as para formar uma lista ordenada completa. O Merge Sort é conhecido por ter uma complexidade de tempo garantida de O(n log n), tornando-o uma escolha eficaz para ordenar listas grandes.


 Aqui está uma explicação passo a passo da estrutura do Merge Sort:

 Passos do Merge Sort:

 Divisão: A lista não ordenada é dividida em duas partes iguais (ou quase iguais) no meio. Esse processo é repetido até que cada sub-lista contenha apenas um elemento, o que é considerado trivialmente ordenado.

 Ordenação das Sub-listas: Cada sub-lista é ordenada individualmente. Isso pode ser feito recursivamente aplicando o Merge Sort a cada sub-lista.

 Junção (Merge): As sub-listas ordenadas são combinadas (mescladas) para criar sub-listas maiores e ordenadas. A combinação é feita comparando o primeiro elemento de cada sub-lista e selecionando o menor (ou maior, dependendo da ordem desejada) para formar a nova sub-lista ordenada.

 Continuação da Combinação: O processo de combinação continua até que todas as sub-listas sejam mescladas em uma única lista ordenada.


 Vantagens do Merge Sort:

 Complexidade de Tempo Preditível: O Merge Sort possui uma complexidade de tempo garantida de O(n log n) para todos os casos, independentemente da sequência inicial dos elementos, o que o torna eficiente para listas grandes.

 Estabilidade: O Merge Sort é um algoritmo estável, o que significa que ele preserva a ordem relativa de elementos iguais após a ordenação.

 Adaptação: O desempenho do Merge Sort não é afetado pela distribuição inicial dos elementos. Ele sempre divide as listas pela metade, independentemente dos valores dos elementos.


 Desvantagem do Merge Sort:

 Uso de Memória Adicional: O Merge Sort geralmente requer espaço de memória adicional para armazenar as sub-listas durante o processo de divisão e mesclagem. Isso pode ser um problema para listas muito grandes ou em sistemas com restrições de memória.


 Ilustração:

 1- Considere a lista não ordenada [5, 2, 9, 1, 5, 6]:

 2- Divisão: A lista é dividida em duas sub-listas [5, 2, 9] e [1, 5, 6].

 3- Ordenação das Sub-listas: As sub-listas são ordenadas individualmente: [2, 5, 9] e [1, 5, 6].

 4- Junção (Merge): As sub-listas ordenadas são mescladas para formar [1, 2, 5, 5, 6, 9], que é a lista ordenada final.


 O Merge Sort é amplamente utilizado em algoritmos de ordenação eficientes e é especialmente útil quando a estabilidade, a previsibilidade de desempenho e a eficiência em listas grandes são prioridades.
